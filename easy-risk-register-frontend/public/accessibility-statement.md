# Accessibility Statement for Easy Risk Register

**Effective Date:** November 15, 2025

## Our Commitment to Accessibility

Easy Risk Register is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards as defined by the Web Content Accessibility Guidelines (WCAG) 2.1 AA.

## Measures to Support Accessibility

Easy Risk Register takes the following measures to ensure accessibility:

- Include accessibility as part of our design and development process
- Provide accessibility training for our development team
- Integrate automated accessibility testing in our development workflow
- Regular accessibility reviews and testing

## Conformance Status

The web application Easy Risk Register is fully conformant with WCAG 2.1 AA accessibility standards. The application has been tested using automated tools (axe-core) and manual review to ensure compliance.

## Technical Specifications

The application uses the following technologies to achieve conformance:

- HTML5
- CSS3
- JavaScript (ES6+)
- React 19
- TypeScript
- Tailwind CSS

## Compliance Status

### WCAG 2.1 AA Guidelines Compliance:

#### Perceivable:
- ✅ Color is not used as the only visual means of conveying information
- ✅ Sufficient contrast ratios (4.5:1 for normal text, 3:1 for large text)
- ✅ Text can be resized up to 200% without loss of functionality
- ✅ Alternative text provided for all meaningful images
- ✅ Headings are structured logically (H1, H2, H3, etc.)

#### Operable:
- ✅ All functionality available from a keyboard
- ✅ Sufficient time to read and use content
- ✅ No content that causes seizures
- ✅ Navigable in a logical order with links to skip content
- ✅ Consistent identification for repeated components
- ✅ Sufficient contrast for UI components and status indicators

#### Understandable:
- ✅ Text is readable and understandable
- ✅ User interface operations are predictable
- ✅ Input assistance provided for forms and errors
- ✅ Status messages are properly announced to screen readers

#### Robust:
- ✅ Compatible with various assistive technologies
- ✅ Valid HTML structure
- ✅ Proper ARIA attributes and roles

## Known Limitations

While we strive to maintain full compliance with WCAG 2.1 AA standards, we acknowledge:

- Some complex visualizations may require additional description for screen readers
- Dynamic content updates are continuously being improved for screen reader compatibility

## Assessment Approach

Easy Risk Register has been assessed for conformance to WCAG 2.1 AA standards using:

- Automated testing tools (axe-core)
- Manual keyboard navigation testing
- Screen reader testing (NVDA, JAWS)
- Color contrast analysis
- Focus indicator visibility testing

## Additional Considerations

To optimize the accessibility experience for all users, particularly those using assistive technologies, we recommend:

- Keeping your browser and assistive technology up to date
- Using the latest stable versions of browsers for the best compatibility
- Adjusting browser settings for optimal text and color preferences
